## 요청배경

25년 10월 선행 아사나 건으로 POS 매출전표 > 결제수단별 매출/취소현황에서 당일 취소된 결제를 미반영(N)하는 정책으로 변경되었다. 그러나 매장관리앱은 당일 발생한 결제와 취소를 모두 포함하여 집계하므로, 결제일과 취소일이 다른 경우 두 시스템 간 매출 수치에 차이가 발생한다. 삼성서울병원과 드림하이테크는 운영 방식 차이로 반영(Y)으로 원복을 요청하였으나, 현재 매장별 개별 설정 수단이 없어 대응이 불가한 상태이다.

## 요청사항

매장관리 > 매장운영설정 > 기타 정책 설정에 "POS 매출전표 당일 취소결제 반영여부" 항목을 추가한다. 라디오 버튼으로 반영(Y) / 미반영(N)을 선택할 수 있게 하며, 기본값은 미반영(N)이다.

## 협의 결론

| 항목 | 내용 |
|------|------|
| 추가 위치 | 매장관리 > 매장운영설정 > 기타 정책 설정 |
| UI | 라디오 버튼 (반영 / 미반영) |
| 기본값 | 미반영(N) |
| 운영 조치 | 삼성서울병원, 드림하이테크 → 반영(Y) 설정 |

## 이슈 원인 (분석 by 정용님)

- POS 매출전표(영수증 매출현황): 해당일에 결제되고, 취소가 안 된 결제의 합
- 매장관리앱: 해당일에 결제, 취소된 결제의 합

결제일과 취소일이 다를 때 두 시스템의 집계 기준이 달라 수치 차이가 발생한다.
