## 2. 마스킹 유틸리티 현황

### 파일: `src/utils/formats/phone.ts`

| 함수명 | 동작 | 마스킹 여부 |
|--------|------|-------------|
| `phoneFormat(input)` | 전화번호 포맷팅만 (하이픈 추가) | 마스킹 없음 (010-1234-5678) |
| `phoneFormatForCommonTable(input)` | 전화번호 포맷팅만 | 마스킹 없음 (010-1234-5678) |
| `phoneFormatForCommonTable(input, true)` | 중간자리 마스킹 | 마스킹 있음 (010-****-5678) |

핵심: `phoneFormatForCommonTable`의 두 번째 인자 `useMasking`을 `true`로 넘겨야만 마스킹이 적용됨.

## 5. 수정 방법

모든 수정은 기존 유틸리티 `phoneFormatForCommonTable`의 `useMasking` 인자를 `true`로 전달하는 것으로 통일 가능.

```typescript
// Before (마스킹 없음)
mobileNo                                    // 원문 그대로 (01012345678)
phoneFormat(mobileNo)                       // 포맷팅만 (010-1234-5678)
phoneFormatForCommonTable(mobileNo)          // 포맷팅만 (010-1234-5678)

// After (마스킹 적용)
phoneFormatForCommonTable(mobileNo, true)    // 마스킹 적용 (010-****-5678)
```

`phoneFormatForCommonTable`을 import하지 않은 파일에서는 import 추가 필요:
```typescript
import { phoneFormatForCommonTable } from '@/utils/formats/phone'
```

## 6. 참고: 엑셀 다운로드 시 전화번호 미포함 확인 완료된 화면

아래 엑셀 다운로드 핸들러들은 전화번호 관련 필드가 없음을 확인함:

- 다중사업자매출내역 (`MultiStoreSalesHistory`)
- 다중사업자결제내역 (`MultiStorePaymentHistory`)
- 컵보증금 거래내역 (`CupDepositPaymentHistory`)
- 상품별 매출내역 (`ProductSalesHistory`)
- PG정산내역 (`PgSettlementHistory`)
- 부가세 신고자료 (`VatReport`)
- 매장 쿠폰팩 (`Store/Event/StoreCouponPack`) -- 전화번호 미포함
- 직원포인트 지급/사용이력 (`EmployeePay`)
- 기타 SalesStats (일별/요일별/시간대별/월별/상품별 매출통계 등) -- 전화번호 미포함
- 상품/재고 관리 관련 엑셀 -- 전화번호 미포함
- 다국어 관리, 브랜드 관리 -- 전화번호 미포함
