# 엑셀 다운로드 전화번호 마스킹 전수 재분석

> 분석일: 2026-03-27
> 대상: bw_frontend_backoffice 전체

## 검증 방법

1. `ExcelDownloadButton` 컴포넌트 사용처 전수 검색 (tsx 파일)
2. 각 사용처에서 `{/* <ExcelDownloadButton` 형태의 JSX 주석 여부를 코드를 열어서 직접 확인
3. `excelDownload2` / `excelDownload` 함수 직접 호출처도 별도 검색
4. 활성화된 엑셀 다운로드만 대상으로 엑셀에 포함되는 컬럼/필드에서 전화번호 관련 데이터 포함 여부 확인
5. 전화번호가 포함된 경우, 마스킹 처리 여부 확인:
   - `phoneFormatForCommonTable(value, true)` = 마스킹 적용 (중간자리 ***)
   - `phoneFormatForCommonTable(value)` 또는 `phoneFormatForCommonTable(value, false)` = 마스킹 미적용 (전체 노출)
   - `phoneFormat(value)` = 마스킹 미적용 (하이픈만 추가)
   - `mobileNo` 직접 사용 = 마스킹 미적용 (원본 그대로)

## 요약

- **마스킹 미적용 (수정 필요)**: 10건 (대표번호 1건 제외 시 10건, 포함 시 11건)
- **마스킹 적용 완료**: 3건 (알림톡 발송이력 계열)
- **주석 처리 (현재 미사용)**: 4건 (향후 활성화 시 마스킹 필요)

### 수정 우선순위 제안

1. **최우선 (개인 전화번호 직접 노출)**: #1, #2, #6, #9, #10 - `mobileNo` 원본 또는 명시적 `false`
2. **우선 (포맷팅만 적용)**: #3, #8 - `phoneFormat`으로 하이픈만 추가, 마스킹 없음
3. **일반 (useMasking 미전달)**: #4, #5, #7 - `phoneFormatForCommonTable` 사용하나 `true` 미전달
4. **별도 판단 필요**: #11 - 사업장 대표번호 (개인정보 아닐 수 있음)

### 수정 방법

모든 건의 수정 방법은 동일: 엑셀 다운로드 핸들러에서 전화번호 포맷 함수 호출 시 `phoneFormatForCommonTable(value, true)`로 변경

```typescript
// Before (마스킹 미적용)
mobileNo: phoneFormat(mobileNo)                           // 하이픈만
mobileNo: phoneFormatForCommonTable(item.mobileNo)         // useMasking 미전달
mobileNo: phoneFormatForCommonTable(item.mobileNo, false)  // 명시적 false
mobileNo: item.mobileNo                                    // 원본

// After (마스킹 적용)
mobileNo: phoneFormatForCommonTable(item.mobileNo, true)   // 010-****-1234
```
