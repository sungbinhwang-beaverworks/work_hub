## 배경

더본 티오더 사용 매장에서 비버 포스 교체 시 T-order 서버와 서버 to 서버 연동이 필요하다. T-order는 HTTPS POST, JSON, api-key 인증 방식의 이벤트 API를 제공하며, 비버 포스에서 상품/매출/테이블 등 변경사항이 발생하면 T-order 서버로 callback 이벤트를 전달하는 구조이다.

## 이번 이슈에서 해야 할 것

### 1. 연동설정 모달 추가
- 위치: 매장관리 > 매장운영설정 > 매장연동 정보 > 외부연동 현황 > 티오더 행 > "연동설정" 버튼
- 클릭 시 "T-order 연동설정" 모달 열림
- 참고 패턴: 식권대장(MealKingSettingModal) — API 키 인증형, 심플한 구조
- 모달 입력 항목: API Key (필수), 매장 식별 코드 / Store ID (필수)
- 공통코드(EXTNL_ITLOK_TGT_CD)에서 티오더의 etc3='Y' 또는 'M' 설정 필요

### 2. 상품연동 버튼 추가
- 이상환님: "상품정보 변경에 대한 부분은 별도 버튼으로 진행"
- 외부연동 현황 테이블 티오더 행에 "상품연동" 버튼 추가
- 클릭 시 현재 매장의 전체 상품/카테고리 정보를 T-order 서버에 전달
- 공통코드에서 티오더의 etc1='Y' 설정 필요
- 기존 etc1='Y' 상품연동 버튼 패턴과 동일한 위치

### 3. 매장 개점/마감 연동 제외
- 이상환님: "매장 개점/마감에 대한 연동 제외"
- API 명세서에도 TBD 상태
- 마감 상태에서 주문 인입 시 당일자로 자동개점되면서 매출적재 — 서버 로직으로 처리

## 코드 분석 결과

### 외부연동 현황 버튼 표시 로직 (StoreInterfaceCard.tsx)
- 공통코드(EXTNL_ITLOK_TGT_CD)의 etc1='Y' → 상품연동 버튼 표시
- 공통코드의 etc3='Y' 또는 'M' → 연동설정 버튼 영역 표시
- 연동대상 코드별로 분기하여 각 전용 모달 렌더링
- 티오더는 현재 분기가 없어서 793줄 else { return <></> }에 빠짐 → 버튼 미표시

### 기존 연동설정 모달 패턴 (참고용)
- 서버 연결형 (IP/Port): OK POS, 이지포스, 아스템즈
- API 키 인증형: 식권대장 (Client ID/Secret), 비플페이 (가맹점코드/인증키/암호화 인증키)
- T-order는 API Key 인증 방식이므로 식권대장 패턴이 가장 유사

### 상품연동 버튼 (두 곳에 존재)
1. 외부연동 현황 테이블 행 우측 (StoreInterfaceCard.tsx) — etc1='Y'일 때
2. 매장관리 우측 상단 (LayoutHeader.tsx) — extnlItlokNo가 있을 때
- 둘 다 동일한 API(useMutationStmExtnlItlokProduct) 호출

## 이상환님 요구사항 정리

1. **callback API 방식**: T-order 이벤트 API(HTTPS POST, api-key 인증)로 상품/매출 변경사항 전달
2. **별도 상품연동 버튼**: "상품정보 변경에 대한 부분은 별도 버튼으로 진행" — 기존 상품연동 버튼 패턴(etc1='Y')과 동일 위치
3. **매장 개점/마감 연동 제외**: 마감 상태에서 주문 인입 시 당일자로 자동개점 + 매출적재는 서버 로직

## 티오더 이벤트 API 명세서 (v1.3.0)

| Event Code | Description | 비고 |
|-----------|-------------|------|
| P0101 | 포스 프로그램 시작 | 자동 전달 |
| P0102 | 포스 프로그램 종료 | 자동 전달 |
| P0201 | 테이블 정보 변경 | 자동 전달 |
| P0301 | 상품 그룹 정보 변경 | 상품연동 callback — 별도 버튼 |
| P0302 | 상품 정보 변경 | 상품연동 callback — 별도 버튼 |
| P0401 | 주문서 정보 변경 | 매출 연동 — 자동 전달 |
| P0402 | 테이블 정리 | 자동 전달 |
| P0501 | 결제완료 처리 (후불제) | 매출 연동 — 자동 전달 |

- 통신: HTTPS POST, JSON, api-key 인증
- 매장 개점/마감: TBD → 이상환님 요청으로 제외
