# T-order 연동설정 기획 얼개

## 1. 개요

더본 티오더 사용 매장에서 비버 포스 교체 시, T-order 서버와의 연동 설정을 매장관리 백오피스에서 관리할 수 있어야 한다.

적용 위치: 매장관리 > 매장운영설정 > 매장연동 정보 > 외부연동 현황 > 티오더 행 (외부연동번호 11403)

현재 상태: 티오더가 외부연동번호 11403으로 등록되어 있으나, 연동설정 버튼과 상품연동 버튼이 없다. 기존 식권대장, 비플페이 등 다른 외부연동과 동일한 패턴으로 버튼과 모달을 추가한다.

## 2. 연동설정 모달

외부연동 현황 테이블의 티오더 행에 "연동설정" 버튼을 추가한다. 클릭 시 "T-order 연동설정" 모달이 열린다.

T-order는 HTTPS API Key 인증 방식의 서버 to 서버 연동이므로, IP/Port 입력이나 장비 매핑이 필요 없다. 기존 모달 중 식권대장(Client ID/Secret만 입력하는 심플한 구조)과 가장 유사하다.

### 모달 입력 항목

| 필드명 | 타입 | 필수 | 설명 |
|--------|------|------|------|
| API Key | Input | Y | T-order에서 발급받은 api-key 인증값 |
| 매장 식별 코드 (Store ID) | Input | Y | T-order 측에서 부여한 매장 코드 |

하단에 초기화 / 적용 버튼을 배치한다.

## 3. 상품연동 버튼

외부연동 현황 테이블의 티오더 행에 "상품연동" 버튼을 추가한다.

클릭 시 현재 매장의 전체 상품/카테고리 정보를 T-order 서버에 전달한다. 모달 없이 바로 동작하며, 기존 아스템즈 등의 상품연동 버튼과 동일한 위치와 패턴이다.

상품 정보 변경에 대한 부분을 별도 버튼으로 진행한다는 이상환님 요구사항을 반영한 것이다.

## 4. 제외 사항

- 매장 개점/마감 연동은 제외한다. (이상환님 요청, API 명세서에서도 TBD 상태)
- 마감 상태에서 주문 인입 시 당일자로 자동개점되면서 매출적재되는 동작은 서버 로직으로 처리한다.

## 5. 협의 필요 사항

- 상품연동 버튼 클릭 시 기존 상품연동 API를 재사용할 수 있는지, T-order 전용 API가 필요한지
- 연동설정 모달의 입력 항목이 API Key와 Store ID 외에 추가로 필요한 것이 있는지
- 매출 연동(P0401/P0501)은 연동 설정 완료 후 자동으로 동작하는 것인지, 별도 설정이 필요한지
