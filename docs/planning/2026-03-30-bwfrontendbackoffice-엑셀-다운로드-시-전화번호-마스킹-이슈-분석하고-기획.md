# 기획 문서: bw_frontend_backoffice 엑셀 다운로드 시 전화번호 마스킹 적용

## 1. 문제 정의
`bw_frontend_backoffice` 프로젝트에서 회원 정보 엑셀 다운로드 시 전화번호(`mobileNo`) 필드가 마스킹 처리되지 않고 원본 그대로 노출되어 개인 정보 보호에 취약한 문제가 있습니다.

## 2. 변경 범위

| 카테고리     | 대상                                         | 변경 내용                                                                                                                                                                                                                                                                       | 근거                                                                                                                                           |
| :----------- | :------------------------------------------- | :---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | :--------------------------------------------------------------------------------------------------------------------------------------------- |
| **코드 로직** | **엑셀 다운로드 로직 (구현 파일 미확인)**    | 엑셀 파일을 생성하는 로직 내에서 `mobileNo` 필드 데이터를 엑셀에 작성하기 전, 마스킹 유틸리티를 적용하여 마스킹된 형태로 엑셀에 출력합니다. (현재 분석 대상 파일에는 해당 구현 로직이 없으므로, **해당 로직 파일을 먼저 식별하는 것이 필수적입니다.**)                                | 분석 보고서에서 `exceljs` 라이브러리 사용 확인 및 엑셀 다운로드 시 마스킹 누락 보고 (핵심 발견 사항, 권고 1), 엑셀 다운로드 로직 분석의 필요성 강조 (제한 사항) |
|              | **프론트엔드 마스킹 유틸리티 (신규 구현)**   | `mobileNo`를 표준 마스킹 정책에 따라 처리하는 재사용 가능한 유틸리티 함수를 신규 구현합니다. (예: `maskPhoneNumber(mobileNo: string): string`) 이는 엑셀 다운로드뿐 아니라 `mobileNo`를 사용하는 다른 프론트엔드 화면에도 적용 가능성을 열어둡니다. | 분석 보고서의 권고 3, 일관된 마스킹 처리 및 재활용성 확보                                                                                      |
| **정책/가이드** | **전화번호 마스킹 정책 정의 (신규 정의)**    | 전화번호 마스킹 형식에 대한 구체적인 정책을 정의합니다. (예: `010-****-1234`, `010-XXXX-1234` 등)                                                                                                                                                                             | 분석 보고서의 권고 2, 개발 진행 전 표준화된 마스킹 규칙 필요                                                                                   |
| **영향 없음** | **API 정의 파일 및 응답 모델 (`src/apis/*`)** | 본 기획은 프론트엔드 단에서의 마스킹 처리를 목표로 하므로, `src/apis/*` 에 정의된 API 응답 모델의 `mobileNo` 필드 타입 또는 백엔드 응답 로직은 변경하지 않습니다. 단, 향후 API 레벨에서의 마스킹 정책이 수립될 경우 별도 기획이 필요합니다.           | 분석 보고서에서 `mobileNo`가 `string` 타입으로 정의되어 있으며, API 응답 변경은 본 기획 범위 외 (권고 4는 '선택적' 검토 사항으로 명시)              |
