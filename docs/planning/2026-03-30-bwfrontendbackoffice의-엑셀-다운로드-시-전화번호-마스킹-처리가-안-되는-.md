# 기획 문서: bw_frontend_backoffice 엑셀 다운로드 시 회원 전화번호 마스킹 미처리 이슈 대응

## 1. 문제 정의
`bw_frontend_backoffice` 시스템에서 회원 정보를 엑셀로 다운로드할 때, 전화번호(`mobileNo`)가 마스킹 처리되지 않은 채로 노출되어 개인정보 보호 규정 위반의 소지가 있습니다. 이는 클라이언트 측 API 정의 파일에서 `mobileNo` 데이터에 대한 마스킹 로직이 부재하여 백엔드로부터 수신한 비마스킹 데이터를 엑셀 다운로드 기능에서 그대로 사용하기 때문에 발생합니다.

## 4. 검증 방법

1.  **시각적 검증 (수동 테스트)**
    *   **대상**: 변경 범위에 포함된 4가지 API를 통해 데이터를 조회하고 엑셀 다운로드 기능을 제공하는 모든 화면
    *   **절차**:
        *   각 대상 화면에서 엑셀 다운로드 버튼을 클릭하여 파일을 저장합니다.
        *   다운로드된 엑셀 파일을 열어 `mobileNo` 또는 전화번호 관련 컬럼을 확인합니다.
    *   **기대 결과**: 모든 `mobileNo` 데이터가 정의된 마스킹 패턴(`010-****-1234` 등)에 따라 올바르게 처리되어야 합니다. 마스킹되지 않은 전화번호는 발견되어서는 안 됩니다. (근거: 마스킹 유틸리티 함수 적용)

2.  **데이터 무결성 검증**
    *   **대상**: 마스킹 로직이 적용된 엑셀 데이터 생성 파이프라인
    *   **절차**:
        *   마스킹 전후의 데이터 스냅샷을 비교할 수 있는 환경이라면, 특정 데이터를 엑셀로 다운로드한 후 원본 데이터와 비교하여 마스킹된 필드 외 다른 필드의 데이터가 변조되지 않았음을 확인합니다.
    *   **기대 결과**: `mobileNo` 필드만 마스킹 처리되고, 그 외의 다른 모든 필드의 데이터는 원본과 동일하게 유지되어야 합니다. (근거: 특정 필드에 대한 선택적 변환 로직 적용)

3.  **단위 테스트 (선택 사항, 코드 품질 향상)**
    *   **대상**: 신규로 개발되는 `maskPhoneNumber` 유틸리티 함수
    *   **절차**:
        *   다양한 형식의 전화번호 (유효/유효하지 않은, 짧은/긴 번호 등)를 입력 값으로 하여 `maskPhoneNumber` 함수의 반환 값을 검증하는 단위 테스트 코드를 작성 및 실행합니다.
    *   **기대 결과**: 유효한 전화번호는 올바르게 마스킹되고, 유효하지 않은 전화번호나 예외 케이스는 예상대로 처리되어야 합니다. (근거: `maskPhoneNumber` 함수의 신뢰성 확보)
