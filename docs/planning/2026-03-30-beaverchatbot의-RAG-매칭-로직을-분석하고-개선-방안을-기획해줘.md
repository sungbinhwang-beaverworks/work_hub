# 기획 문서: beaver_chat_bot RAG 매칭 로직 개선 (초안)

## 1. 문제 정의

사용자 원문에서 "챗봇의 RAG 매칭이 부정확한 경우가 있어서 분석 후 기획이 필요함"이라는 문제가 제기되었습니다. 이 문제를 해결하기 위한 기획을 수립하고자 하였으나, 현재 제공된 분석 보고서는 분석 대상 파일 및 데이터 현황만을 포함하며, **RAG 매칭이 부정확하게 발생하는 구체적인 원인과 그에 대한 상세 분석 결과가 부재합니다.** 따라서 현재 단계에서는 문제의 근본 원인을 명확히 정의하기 어렵습니다.

**정확한 문제 정의를 위한 추가 분석 필요 사항:**

*   **부정확한 매칭 사례 구체화:** 어떤 사용자 질문에 대해 어떤 매뉴얼 청크가 매칭되었고, 이것이 왜 부정확하다고 판단되는지 (예: 관련성 낮음, 의도와 동떨어진 정보 제공 등) 최소 3개 이상의 구체적인 사례를 통해 설명이 필요합니다. 이러한 사례는 문제의 실체와 사용자 경험에 미치는 영향을 명확히 보여줄 것입니다.
*   **현재 RAG 매칭 로직의 한계점:** `src/app/api/chat/route.ts` 내 RAG 검색, 키워드 추출, 매뉴얼 매칭 로직의 각 단계에서 발생할 수 있는 문제점 (예: 키워드 추출 정확도 부족, 벡터 유사도 임계치 부적절, `modes` 필터링 로직의 오작동 등)에 대한 분석 결과가 필요합니다. 이는 문제의 기술적 원인을 파악하는 데 필수적입니다.
*   **데이터 관점 분석:** `supabase/schema.sql`의 `match_chunks` 함수 정의 및 `scripts/ingest.ts`의 `modes` 필드 처리 방식이 현재 문제에 미치는 영향에 대한 분석이 필요합니다. 또한, `chunks` 테이블의 데이터 구조나 내용 (예: 청크의 길이, 중복성, 핵심 정보 포함 여부)이 매칭 정확도에 영향을 주는지 여부도 포함되어야 합니다. 이는 데이터 구조적 관점에서의 개선 가능성을 탐색하는 데 중요합니다.

## 2. 변경 범위

현재 단계에서는 RAG 매칭의 부정확성을 야기하는 근본 원인이 파악되지 않았으므로, 어떠한 파일/기능이 영향을 받아야 할지 또는 어떤 모듈을 변경해야 할지 결정할 수 없습니다.

**변경 범위 결정을 위한 추가 분석 필요 사항:**

*   **문제 원인 식별:** 문제의 원인이 특정 로직(예: 키워드 추출 로직, 벡터 검색 매개변수), 데이터 처리 방식(예: 청크 변환 방식, `modes` 필드 인덱싱), 혹은 데이터 자체(예: 매뉴얼 내용의 품질) 중 어디에 있는지 명확히 파악되어야 합니다. 이를 통해 변경이 필요한 시스템의 영역을 특정할 수 있습니다.
*   **예상 개선 방안 도출:** 분석을 통해 도출된 잠재적인 개선 방안(예: 키워드 추출 로직 변경, 벡터 검색 모델/임베딩 변경, 청크 전처리 강화, 필터링 로직 수정 등)에 따라 변경될 핵심 파일 및 기능 목록이 식별되어야 합니다. 예를 들어, 키워드 추출 로직 문제라면 `src/app/api/chat/route.ts` 내 특정 함수가 변경 범위에 포함될 것이며, 데이터 인제스트 문제라면 `scripts/ingest.ts`가 변경 범위에 포함될 수 있습니다.

## 4. 검증 방법
