## 2. 현재 코드 상태

### 동작하는 것
- PixiJS 2D 캔버스 렌더링 (탑다운 32x32 격자)
- 에이전트 캐릭터 스프라이트 (Gather Clone Character_001/004/010)
- 가구 스프라이트 배치 (Pixel Agents: 책상, PC, 의자, 화분 등)
- 드래그 패닝 + 마우스 휠 줌
- 채팅 오버레이 + HUD 연동
- Supabase Realtime 구독
- SSE 스트리밍 채팅

### 문제점 (다음 세션에서 해결)
- **바닥이 단색 사각형** — 야외 타일셋(stone_wall 등) 적용 시도했으나 원시부족 느낌
- **벽이 1px 선** — 두께감 있는 벽이 필요
- **가구 배치가 어지러움** — 정리 필요 (현재는 정리된 상태)
- **방이 사무실처럼 안 보임** — 바닥 텍스처 + 벽 두께가 핵심

## 6. 핵심 학습

- ZEP/Gather는 **2D 탑다운 직교 격자**(32x32)에 스프라이트를 비스듬히 그려서 깊이감을 줌. 좌표계 자체는 아이소메트릭이 아님.
- 시각적 퀄리티는 코드가 아니라 **스프라이트 에셋**이 결정함. 에셋 없이 코드만 바꿔도 메모장.
- 디자인 작업은 용어 설명보다 **레퍼런스 이미지를 먼저 확인**하는 게 핵심.
- `reactStrictMode: false` 설정 필수 (WebGL Context Lost 방지).
