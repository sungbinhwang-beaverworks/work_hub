# 분석 보고서: bw_frontend_backoffice 엑셀 다운로드 회원 전화번호 마스킹 미처리 이슈

## 1. 분석 대상

*   **프로젝트**: bw_frontend_backoffice
*   **파일**: 총 51개 파일 중 `src/apis` 경로 하위의 API 정의 파일 및 `package.json`, `src/App.tsx` 파일
*   **주제**: 엑셀 다운로드 시 회원 전화번호 마스킹 미처리 현상 분석

## 2. 현황

제공된 `bw_frontend_backoffice` 프로젝트 코드 분석 결과, 엑셀 다운로드 기능 구현에 사용되는 라이브러리가 확인되었으며, 다수의 API 응답 모델에서 마스킹되지 않은 전화번호 필드가 정의되어 있음을 확인하였습니다. 사용자 이슈에 따르면, 이러한 필드들이 엑셀 다운로드 과정에서 마스킹 처리 없이 노출되는 것으로 판단됩니다.

## 3. 발견 사항

### 3.1 엑셀 다운로드 관련 라이브러리

`package.json` 파일에서 다음의 엑셀 처리 라이브러리 의존성이 확인됩니다. 이는 프로젝트 내에 엑셀 다운로드 기능이 구현되어 있음을 시사합니다.

*   `exceljs`: 버전 4.4.0 (1개 확인)
*   `xlsx`: 버전 0.18.5 (1개 확인)

### 3.2 전화번호 필드를 포함하는 API 엔드포인트

`src/apis` 디렉토리 내에서 회원 또는 상점의 전화번호 정보를 직접적으로 반환하는 API 엔드포인트 5개가 확인되었습니다. 이들 API는 모두 `string` 타입으로 전화번호를 반환하며, 코드 상에서 명시적인 마스킹 처리는 없습니다.

| API 경로                              | 관련 데이터 타입    | 전화번호 필드명 | 설명                  | 응답 형태  |
| :------------------------------------ | :------------------ | :-------------- | :-------------------- | :--------- |
| `/bw/cntr/contract/list`              | `CntrInfo`          | `mobileNo`      | 사용자 전화번호       | `string`   |
| `/bw/cntr/install/list`               | `InstlInfo`         | `mobileNo`      | 사용자 전화번호       | `string`   |
| `/bw/cntr/user/list`                  | `UserListData`      | `mobileNo`      | 대표자 전화번호       | `string`   |
| `/bw/cntr/user/totlist`               | `UserInfo`          | `mobileNo`      | 회원 전화번호         | `string`   |
| `/bw/cntr/store/list`                 | `StoreInfo`         | `bsplcTelNo`    | 상점 전화번호         | `string`   |

### 3.3 데이터 처리 방식

*   모든 API는 `useQuery` 또는 `useMutation` 훅을 통해 `@tanstack/react-query` 라이브러리를 사용하여 데이터를 비동기적으로 조회 및 변형합니다.
*   `queryGeneric` 함수 또는 직접적인 `fetch` API 호출을 통해 `POST` 메소드로 백엔드(`process.env.REACT_APP_BWB_API_URL`)에 요청하며, `Content-Type: application/json` 헤더와 JSON 형태의 `body`를 사용합니다.
*   Hmac 인증이 필요한 API 호출의 경우 `GenerateHmac` 함수를 사용하여 `Hmac` 헤더를 추가합니다. `src/apis/cntr/store/classInquiry.ts`, `src/apis/com/multi/lang/bundle.ts`, `src/apis/com/user/autoLoginCheck.ts`, `src/apis/com/user/login.ts` 4개 API는 Hmac 인증을 사용하지 않습니다.
*   API 응답은 `CommonResponseType`를 상속하며, 전화번호 필드는 백엔드로부터 마스킹되지 않은 원본 `string` 형태로 전달되는 것으로 확인됩니다.
*   현재 제공된 파일 내에서는 엑셀 다운로드 로직 자체는 포함되어 있지 않으므로, 데이터가 API로부터 프론트엔드로 전달된 후 엑셀 파일로 변환되는 과정에서 마스킹이 누락된 것으로 추정됩니다.

## 4. 권고

1.  **전화번호 마스킹 정책 정의 및 적용**: 개인 식별 정보를 포함하는 전화번호는 보안 및 개인정보 보호를 위해 반드시 마스킹 처리되어야 합니다. 마스킹 방식(예: `010-****-1234` 또는 `010-1234-****`)에 대한 명확한 정책을 수립하고, 이를 엑셀 다운로드 기능에 적용해야 합니다.
2.  **프론트엔드 엑셀 생성 로직 검토**: 엑셀 파일 생성 시 전화번호 필드에 대한 마스킹 로직이 누락된 것으로 판단됩니다. `exceljs` 또는 `xlsx`를 사용하는 엑셀 생성 관련 유틸리티 함수나 컴포넌트 내에서, 상기 5개 API(및 잠재적으로 다른 전화번호를 포함하는 API)로부터 받은 데이터를 엑셀에 쓰기 전에 마스킹 처리를 적용해야 합니다.
3.  **마스킹 유틸리티 통합**: 여러 위치에서 전화번호 마스킹이 필요할 수 있으므로, 재사용 가능한 공통 마스킹 유틸리티 함수를 구현하여 일관된 마스킹 처리를 보장할 것을 권고합니다.
4.  **백엔드 API 마스킹 검토**: 현재 백엔드 API는 마스킹되지 않은 전화번호를 반환합니다. 만약 백엔드에서부터 마스킹된 데이터를 제공하는 것이 더 적절한 보안 정책이거나, 데이터 사용처에 따라 마스킹 수준이 달라질 필요가 있다면 백엔드 API의 응답 데이터 마스킹 여부도 추가로 검토할 필요가 있습니다.

## 5. 제한 사항

*   제공된 파일 내에서 실제 엑셀 다운로드를 트리거하고 파일을 생성하는 구체적인 프론트엔드 로직 (예: `ExcelService.ts` 또는 특정 페이지 컴포넌트)은 포함되어 있지 않습니다. 따라서 마스킹 누락이 발생한 정확한 코드 위치는 추가 확인이 필요합니다.
*   런타임 동작이 아닌 정적 코드 분석에 기반한 결과입니다.
*   `src/apis` 디렉토리 내에서 `mobileNo` 또는 `bsplcTelNo`와 같은 명시적인 전화번호 필드명을 기준으로 분석하였습니다. 다른 명칭으로 전화번호를 취급하는 필드가 있을 경우 추가 확인이 필요합니다.
