## 1. 제품 개요

### 한 줄 정의
AI 에이전트 팀이 **자율적으로 업무를 수행하는 시스템** + 그 과정을 시각적으로 보고 제어하는 2D 가상 오피스 웹앱.

### 핵심 구조

```
manual_automation/                ← 이게 진짜 허브 (전체 작업 환경)
├── manual_builder_dev/           ← 매뉴얼 빌더 프로젝트
├── beaver_chat_bot/              ← 챗봇 프로젝트
├── bw_frontend_backoffice/       ← 백오피스 프론트엔드
├── work_hub/                     ← 에이전트 허브 + 웹 인터페이스
│     ├── src/                    ← 웹앱 (모니터링/제어 UI)
│     ├── agents/                 ← 에이전트 코어 (자율 업무 엔진)
│     └── docs/                   ← 에이전트 산출물 (분석/기획/설계 문서)
└── ...기타 프로젝트들

[외부 도구] ──MCP 플러그인──→ [에이전트 코어] ──→ [웹앱으로 시각화]
  Asana                          분석/기획/설계        2D 오피스
  Figma                          에이전트 간 협업       모니터링+제어
  Calendar                       프로젝트 폴더 접근
```

### 제품이 아닌 것
- ❌ 예쁜 UI가 목적인 메타버스 서비스
- ❌ 채팅봇 래퍼
- ❌ 대시보드

### 제품인 것
- ✅ 에이전트가 **알아서 일하는 시스템**
- ✅ 사용자는 **모니터링하고 필요할 때 개입**
- ✅ 웹앱은 그 과정을 **보여주는 창문**

---

## 2. 배경과 동기

- 클로드 코드로 실제 업무(분석, 설계, 관리)를 하고 있음
- 이 업무 패턴을 스킬(/analyze, /design, /execute)로 체계화함
- 이제 이 스킬들을 **자동으로 실행하는 에이전트**로 만들고 싶음
- 에이전트들이 일하는 모습을 **ZEP 메타버스 스타일**로 시각화
- 인스타에서 본 "에이전트 팀이 데일리스크럼 하는" 컨셉에서 영감

---

## 3. 타겟 사용자

**나(beaver_bin) 한 명.** 개인 SaaS.

- 제품기획자, 개발자 아님
- 3개 프로젝트를 병행 관리 (매뉴얼빌더, 챗봇, 백오피스)
- AI 에이전트에게 분석/기획/설계 업무를 위임하고 싶음
- 에이전트들이 알아서 일하고, 나는 결과를 확인하고 방향을 잡아주는 역할

---

## 4. 핵심 가치

| 순위 | 가치 | 설명 |
|------|------|------|
| 1 | **자율적 업무 수행** | 에이전트가 트리거를 인식하고 알아서 분석/기획/설계를 진행 |
| 2 | **에이전트 간 협업** | 분석 → 기획 → 설계 순서로 에이전트끼리 업무를 넘김 |
| 3 | **시각적 모니터링** | 에이전트들이 뭘 하고 있는지 2D 오피스에서 한눈에 봄 |
| 4 | **필요 시 개입** | 사용자가 언제든 에이전트에게 말 걸어서 방향 수정 가능 |

## 5. 시스템 레이어

### Layer 1: 에이전트 코어 (두뇌) — 가장 중요

에이전트들이 실제로 일하는 엔진.

| 구성요소 | 역할 |
|---------|------|
| **에이전트 정의 (YAML)** | 각 에이전트의 역할, 페르소나, 프롬프트, 가드레일 |
| **업무 파이프라인** | 트리거 → 인식 → 분석 → 기획 → 설계 → 결과 |
| **에이전트 간 메시지** | 분석 에이전트가 "파악 완료"하면 기획 에이전트에게 전달 |
| **프로젝트 접근** | manual_automation/ 하위 폴더들을 읽고 코드/데이터 파악 |
| **LLM 호출** | Gemini Flash로 분석/판단/생성 |

### Layer 2: 외부 연동 (감각) — MCP 플러그인

에이전트가 외부 세계와 소통하는 창구.

| 플러그인 | 역할 | 예시 |
|---------|------|------|
| **Asana MCP** | 이슈/태스크 트리거 | 새 이슈 발행 → 에이전트 인식 |
| **Figma MCP** | 디자인 참조 | 디자인 스펙 확인, 컴포넌트 체크 |
| **Calendar MCP** | 스케줄링 | 데일리스크럼 시간에 자동 회의 |
| **GitHub MCP** | 코드 변경 감지 | PR 생성, 코드리뷰 (향후) |

### Layer 3: 웹앱 (눈과 손) — 모니터링/제어 UI

2D 가상 오피스 인터페이스.

| 구성요소 | 역할 |
|---------|------|
| **2D 오피스 맵** | 에이전트들이 방에서 일하는 모습 시각화 |
| **에이전트 캐릭터** | 상태 표시 (대기/작업중/오류), 현재 작업 말풍선 |
| **채팅 오버레이** | 에이전트에게 직접 말 걸어서 지시/질문 |
| **활동 로그** | 에이전트들의 업무 히스토리 |

## 6. 에이전트 정의

### 6.1 에이전트 목록

| ID | 이름 | 역할 | 담당 방 | 기반 스킬 |
|----|------|------|--------|----------|
| analyst | 분석관 | 이슈 인식, 코드/현행 분석, 원인 파악 | 분석실 | /analyze |
| planner | 기획자 | 분석 결과 받아서 기획 문서 작성 | 기획실 | (신규) |
| architect | 설계자 | 기획 받아서 기술 스펙/설계 문서 작성 | 설계실 | /design |
| manager | 관리자 | 전체 상황 파악, 우선순위 조정, 스크럼 진행 | 상황실 | /status |

### 6.2 에이전트 업무 흐름 (예시)

```
[Asana에 이슈 발행됨]
    │
    ▼
관리자: "새 이슈 들어왔습니다 — '상품 목록 정렬 버그'. 분석관님 파악 부탁드려요."
    │
    ▼
분석관: bw_frontend_backoffice/ 코드를 읽음
        "SortableTable 컴포넌트의 정렬 로직에 문제가 있네요. 현행은 이렇고..."
        → 분석 결과를 기획자에게 전달
    │
    ▼
기획자: "이 버그를 고치려면 이런 변경이 필요합니다. 스펙 정리할게요."
        → 기획 문서를 설계자에게 전달
    │
    ▼
설계자: "기획 받았습니다. 기술 스펙 작성할게요."
        → 설계 문서 생성 (docs/design/...)
    │
    ▼
관리자: "설계 완료됐습니다. Claude Code로 구현 가능한 상태예요."
        → 사용자에게 알림
```

### 6.3 에이전트 상태

| 상태 | 의미 | 시각 표현 |
|------|------|----------|
| `idle` | 대기 중, 할 일 없음 | 회색 링, 자리에 앉아 있음 |
| `working` | 업무 수행 중 | 녹색 펄스, 말풍선으로 현재 작업 표시 |
| `waiting` | 다른 에이전트의 결과 대기 | 노란색, "기획 결과 대기 중" |
| `error` | 오류 발생 | 빨간 점멸 |
| `meeting` | 회의 중 (회의실) | 회의실로 이동, 발화 중 |

## 11. 비주얼 컨셉 (웹앱 Layer 3)

### 레퍼런스: ZEP 메타버스

- 2D 픽셀아트, 탑다운 직교 격자 (32x32px)
- 밝고 깨끗한 색감, 귀여운 캐릭터 스프라이트

## 13. 성공 기준

### MVP
1. 에이전트에게 "이 코드 분석해줘"라고 하면, manual_builder_dev/ 코드를 읽고 분석 결과를 돌려줌
2. 분석 에이전트의 결과가 기획 에이전트에게 자동 전달됨
3. 웹앱에서 에이전트들이 일하는 모습이 보임 (상태 변화, 말풍선)
4. 사용자가 채팅으로 에이전트에게 개입 가능

### 궁극적 목표
아사나에 이슈가 올라오면, 에이전트 팀이 알아서 분석→기획→설계를 완료하고, 사용자는 결과만 확인하면 되는 상태.
