# 유저플로우 03: 모니터링 플로우

> 관련 PRD: FR-11 (2D 오피스 맵), FR-12 (에이전트 캐릭터), FR-15 (HUD), FR-16 (활동 로그)
> 작성일: 2026-03-27

---

## 1. 개요

사용자가 웹앱에 접속하여, 에이전트들의 현재 상태/진행 중인 업무/히스토리를 확인하고, 필요 시 개입하는 흐름. **"보고 판단하는"** 사용자의 주요 일상 플로우.

---

## 2. 해피패스 (Happy Path)

```
[사용자] 브라우저에서 Work Hub 접속 (localhost:3000 / 배포 URL)
    │
    ▼
[웹앱] 초기 로딩
    ├─ (1) GET /api/agents → 에이전트 목록 + 현재 상태 로드
    ├─ (2) PixiJS 캔버스 초기화 → 2D 오피스 맵 렌더링
    ├─ (3) 에이전트 캐릭터 배치 (상태에 따른 시각 피드백)
    ├─ (4) HUD 렌더링 (활동 에이전트 수, 현재 업무)
    └─ (5) Supabase Realtime 구독 시작 (hub_agents 테이블)
    │
    ▼
[웹앱] 초기 화면 표시 완료 (3초 이내)
    │
    │  사용자가 보는 화면:
    │  ┌──────────────────────────────────────────────┐
    │  │  [HUD] 활동 에이전트: 1/3  현재: 분석관 작업 중   │
    │  │                                              │
    │  │  ┌────────┐ ┌────────┐ ┌────────┐           │
    │  │  │ 분석실  │ │ 설계실  │ │ 상황실  │           │
    │  │  │  🔍    │ │  📐    │ │  📋    │           │
    │  │  │ (녹펄스)│ │ (회색) │ │ (회색) │           │
    │  │  └────────┘ └────────┘ └────────┘           │
    │  │         ─────── 복도 ───────                  │
    │  │  ┌──────────────────────┐                    │
    │  │  │      회의실           │                    │
    │  │  └──────────────────────┘                    │
    │  └──────────────────────────────────────────────┘
    │
    ▼
[사용자] 상태 확인
    ├─ 에이전트 캐릭터의 시각 피드백으로 현재 상태 파악
    │    - 녹색 펄스: 작업 중
    │    - 회색: 대기
    │    - 빨간 점멸: 오류
    ├─ HUD에서 전체 현황 확인
    └─ 말풍선으로 현재 작업 내용 확인 (working 에이전트)
    │
    ▼
[실시간 상태 변경]  (Supabase Realtime)
    │  예: 분석관이 작업 완료 → idle
    │
    ├─ hub_agents 테이블 UPDATE 이벤트 수신
    ├─ 에이전트 캐릭터 시각 피드백 업데이트 (녹색 → 회색)
    ├─ HUD 업데이트 ("활동 에이전트: 0/3")
    └─ 말풍선 제거
    │
    ▼
[사용자] 필요 시 개입
    ├─ 에이전트 클릭 → 채팅 (유저플로우 01 참조)
    └─ 활동 로그 확인 → 과거 업무 히스토리 열람
```

## 3. 분기 (조건)

### 분기 1: 접속 시 에이전트 상태에 따른 초기 표시

```
[웹앱] 초기 로딩 완료
    │
    ├── [모든 에이전트 idle]
    │     └─ HUD: "활동 에이전트: 0/3"
    │        모든 캐릭터 회색
    │        → 사용자가 업무를 지시하거나 외부 트리거를 기다리는 상태
    │
    ├── [1개 이상 에이전트 working]
    │     └─ HUD: "활동 에이전트: N/3 — {에이전트명}: {current_task}"
    │        해당 캐릭터 녹색 펄스 + 말풍선
    │        → 사용자가 진행 상황을 실시간 모니터링
    │
    └── [1개 이상 에이전트 error]
          └─ HUD: "⚠ {에이전트명} 오류 발생"
             해당 캐릭터 빨간 점멸
             → 사용자의 즉각적 확인 필요 → 클릭 시 채팅으로 상황 확인
```

### 분기 2: 카메라 조작

```
[사용자] 2D 오피스에서 시점 조작
    │
    ├── [드래그] → 카메라 패닝 (맵 내 이동)
    │
    ├── [마우스 휠] → 줌 인/아웃 (0.5x ~ 2.5x)
    │
    └── [에이전트 클릭] → 해당 에이전트 위치로 카메라 자동 패닝 + 채팅 열림
```

### 분기 3: 활동 로그 열람

```
[사용자] 활동 로그 패널 열기 (TBD: 버튼 위치)
    │
    ├── [로그 있음]
    │     └─ 시간순 타임라인 표시 (최근 50건)
    │        각 항목: [시각] [에이전트 이름] [액션] [요약]
    │        예: "14:32 분석관 — 분석 완료: bw_frontend_backoffice 정렬 버그"
    │
    └── [로그 없음 (첫 사용)]
          └─ "아직 업무 기록이 없습니다" 안내 표시
```

## 4. 예외 처리

### 예외 1: Empty — 에이전트가 하나도 없음

| 항목 | 내용 |
|------|------|
| **조건** | hub_agents 테이블이 비어 있음 (초기 설정 안 됨), 또는 YAML 파일 없음 |
| **감지** | GET /api/agents 응답에 agents 배열이 빈 배열 |
| **UI 동작** | 2D 오피스는 렌더링하되 캐릭터 없음. HUD에 "에이전트가 등록되지 않았습니다" 표시 |
| **복구 동선** | (1) src/data/agents/*.yaml 확인 (2) SQL 초기 데이터 재실행 (001_hub_schema.sql) |
| **데이터 영향** | 없음 |

### 예외 2: Error — 에이전트 상태 로드 실패

| 항목 | 내용 |
|------|------|
| **조건** | GET /api/agents API 호출 실패 (Supabase 연결 실패, 서버 에러 등) |
| **감지** | API 응답 500 또는 네트워크 에러 |
| **UI 동작** | (1) 2D 오피스는 렌더링 (빈 맵) (2) HUD에 "에이전트 상태를 불러올 수 없습니다" 표시 (3) "다시 시도" 버튼 |
| **복구 동선** | (1) "다시 시도" 클릭 → GET /api/agents 재호출 (2) 3회 실패 시 "서버 연결을 확인해주세요" 안내 |
| **데이터 영향** | 없음 (읽기 전용) |

### 예외 3: Error — Supabase Realtime 연결 끊김

| 항목 | 내용 |
|------|------|
| **조건** | WebSocket 연결 끊김 (네트워크 불안정, Supabase 서버 이슈) |
| **감지** | Supabase Realtime channel의 `CHANNEL_ERROR` / `TIMED_OUT` 이벤트 |
| **UI 동작** | (1) HUD에 "실시간 연결 끊김" 표시 (작은 경고 아이콘) (2) 에이전트 상태가 마지막 수신 상태로 고정 (stale) |
| **복구 동선** | (1) 자동 재연결 시도 (Supabase 기본: 지수 백오프) (2) 재연결 성공 → GET /api/agents로 최신 상태 동기화 (3) 30초 이상 재연결 실패 → "새로고침하세요" 안내 |
| **데이터 영향** | 없음 (상태 불일치만 발생, 데이터 손실 없음) |

### 예외 4: Error — PixiJS 렌더링 실패 (WebGL Context Lost)

| 항목 | 내용 |
|------|------|
| **조건** | 브라우저 탭 장시간 비활성화 후 복귀, 또는 GPU 리소스 부족 |
| **감지** | PixiJS의 `context-lost` 이벤트 |
| **UI 동작** | (1) 캔버스 영역에 "화면을 새로고침합니다" 표시 (2) 자동 reload 시도 |
| **복구 동선** | (1) PixiJS destroy → 재초기화 시도 (2) 실패 시 페이지 새로고침 안내 |
| **방어 조치** | next.config.ts에 reactStrictMode: false 설정 (이미 적용됨) |

### 예외 5: Permission — 미인증 접속 (향후)

| 항목 | 내용 |
|------|------|
| **조건** | 현재는 인증 없음 (개인 SaaS). 향후 배포 시 발생 가능 |
| **현재 대응** | Non-goal (NG-01). 인증 시스템 구현하지 않음 |
| **향후 대응** | 배포 시 Supabase Auth 또는 basic auth로 간단 보호 |

## 5. 상태 흐름 (모니터링 UI)

### 전체 페이지 상태

```
┌──────────────────────────────────────────────┐
│              페이지 상태 머신                   │
├──────────────────────────────────────────────┤
│                                              │
│  [loading] ──로드완료──→ [connected]          │
│      │                      │               │
│   로드실패              Realtime 끊김          │
│      │                      │               │
│      ▼                      ▼               │
│  [load_error]          [disconnected]        │
│      │                      │               │
│   재시도 성공            재연결 성공            │
│      │                      │               │
│      └─────→ [connected] ←──┘               │
│                                              │
└──────────────────────────────────────────────┘
```

### HUD 표시 규칙

| 페이지 상태 | HUD 표시 |
|------------|----------|
| loading | "로딩 중..." |
| connected + 모든 에이전트 idle | "활동 에이전트: 0/{총수}" |
| connected + N개 working | "활동 에이전트: N/{총수} — {첫 번째 working 에이전트}: {task}" |
| connected + error 에이전트 있음 | "⚠ {에이전트명} 오류 발생" (높은 우선순위) |
| disconnected | "실시간 연결 끊김 — 마지막 동기화: {시각}" |
| load_error | "에이전트 상태를 불러올 수 없습니다" + "다시 시도" 버튼 |

## 6. 실시간 동기화 전략

```
[초기 로딩]
    └─ GET /api/agents (REST) → 전체 상태 스냅샷

[이후 업데이트]
    └─ Supabase Realtime 구독 (WebSocket) → 변경분만 수신
         ├─ INSERT: 새 에이전트 추가 (YAML 추가 후 재시작 시)
         ├─ UPDATE: 상태 변경 (idle ↔ working ↔ error)
         └─ DELETE: (사용하지 않음)

[재연결 시]
    └─ GET /api/agents (REST) → 전체 상태 재동기화
         ※ Realtime 끊김 동안 놓친 이벤트를 보정
```
