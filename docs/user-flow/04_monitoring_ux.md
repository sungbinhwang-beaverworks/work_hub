## 1. 문제 정의

### 1.1 현재 상태 (As-Is)

사용자가 에이전트 활동 정보를 확인하려면 **최소 6번 클릭**이 필요하다.

```
HUD "로그" 버튼 클릭 (1)
  → 활동 로그 패널 열림
    → 파이프라인 항목 클릭 (2)
      → ActivityDetailModal 열림
        → 아코디언 단계 클릭 (3)
          → 요약/권고 확인
            → "보기" 버튼 클릭 (4)
              → 산출물 마크다운 로드
                → 스크롤 (5~6)
```

### 1.2 문제 목록

| # | 문제 | 심각도 | 근거 |
|---|------|--------|------|
| P-01 | 6번 클릭해야 정보가 보임 | 높음 | 모니터링 도구인데 사용자가 능동적으로 파고들어야 함 |
| P-02 | 패널 → 모달 이중 레이어 | 높음 | 로그 패널을 열고, 그 위에 모달을 또 여는 구조. 레이어가 쌓이면서 맥락을 잃음 |
| P-03 | 파이프라인과 메시지가 혼재 | 중간 | 로그 패널에서 pipeline 항목과 message 항목이 시간순으로 섞여 스캔 불가 |
| P-04 | 진행 중 vs 히스토리 구분 없음 | 중간 | 지금 돌고 있는 파이프라인과 어제 끝난 파이프라인이 같은 리스트에 존재 |
| P-05 | 산출물 접근이 너무 깊음 | 높음 | 에이전트가 만든 문서(분석/기획/설계)를 보려면 모달 → 아코디언 → "보기" 버튼 → 마크다운 로드 |
| P-06 | 토스트 알림 없음 | 중간 | 파이프라인 완료/에러 시 사용자가 HUD만 보고 인지해야 함 |
| P-07 | PipelineProgress(좌하단)와 ActivityLog(좌상단)가 중복 | 낮음 | 같은 정보를 두 곳에서 다르게 보여줌 |

### 1.3 해결 원칙

1. **정보가 사용자를 찾아간다** -- 모니터링 도구는 Push형이어야 한다
2. **클릭 횟수 최소화** -- 핵심 정보는 2클릭 이내에 도달
3. **레이어는 1개만** -- 패널 또는 모달, 둘 중 하나. 이중 레이어 금지
4. **진행 중인 것과 끝난 것은 분리** -- 시간적 맥락이 다르다
5. **산출물은 1클릭으로** -- 에이전트가 만든 것을 바로 열람

## 2. 설계 결정

### 2.1 구조 결정: 패널 통합형

| 대안 | 설명 | 장점 | 단점 | 선택 |
|------|------|------|------|------|
| A. 패널+모달 유지 | 현행 유지 | 변경 없음 | 6클릭 문제 해결 불가 | |
| B. 패널만 (모달 제거) | 패널 내에서 모든 정보 표시 | 레이어 1개, 단순 | 패널 폭(340px) 안에 상세 정보 넣기 어려움 | |
| C. **활동 패널 확장형** | 패널을 넓히고(480px), 2영역으로 분할. 상단=진행중, 하단=히스토리. 항목 클릭 시 패널 내에서 상세 확장. 모달 제거. | 1레이어, 2클릭, 정보 밀도 높음 | 패널이 넓어서 캔버스 가림 | 채택 |
| D. 전체화면 대시보드 | 별도 /activity 페이지 | 정보 공간 넓음 | 오피스 맥락에서 벗어남 | |

**결정: C안 (활동 패널 확장형)**

### 2.2 세부 결정

| 항목 | 결정 | 이유 |
|------|------|------|
| 패널 폭 | 480px | 산출물 미리보기까지 표시하려면 340px은 부족 |
| 진행 중 영역 위치 | 패널 상단 고정 | 가장 먼저 봐야 할 정보 |
| 히스토리 영역 | 스크롤 가능한 하단 | 과거 기록은 필요할 때 탐색 |
| 상세 보기 | 패널 내 인라인 확장 | 모달 제거. 클릭하면 같은 패널 안에서 확장 |
| 산출물 보기 | 클릭 시 바로 로드 + 인라인 표시 | "보기" 버튼 없이 산출물 파일명 클릭으로 즉시 로드 |
| 좌하단 PipelineProgress | 제거 | 활동 패널 상단 고정 영역이 대체 |
| 토스트 알림 | 추가 | 파이프라인 상태 변경 시 자동 알림 |
| ActivityDetailModal | 제거 | 활동 패널 내 인라인 상세가 대체 |

### 3.2 파이프라인 완료 알림

```
[Realtime] 파이프라인 상태 → completed
    |
    |- (1) 토스트 알림 자동 표시 (우상단, 5초간)
    |     "파이프라인 완료: 상품 목록 정렬 버그 분석"
    |     [결과 보기] [닫기]
    |
    |- (2) 상단 고정 영역에서 완료 표시
    |     타임라인 전체 ✓ + "완료. 총 2분 12초"
    |     산출물 파일명 즉시 표시
    |
    |- (3) 5초 후 상단 고정 영역 → 히스토리로 이동
    |     (새 진행 중 파이프라인이 없으면 상단 고정 영역 비움)
    |
    v
[사용자] 토스트의 "결과 보기" 클릭 (1클릭)
    |
    v
[활동 패널] 열림 + 해당 파이프라인 상세 자동 확장
    |  산출물 파일명 바로 표시
```

### 3.3 히스토리 열람

```
[사용자] HUD "로그" 버튼 클릭 (1클릭)
    |
    v
[활동 패널] 열림
    |- 상단 고정: 진행 중 파이프라인 (있으면)
    |- 하단 스크롤: 히스토리 목록 (파이프라인만, 메시지는 상세 안에)
    |
    v
[사용자] 히스토리 항목 클릭 (2클릭)
    |
    v
[활동 패널] 해당 파이프라인 상세 인라인 확장
    |  - 타임라인 + 단계별 요약 + 산출물 파일명
    |  - 에이전트 간 메시지는 "메시지 보기" 토글로 숨김
```

### 3.4 에러 발생 시

```
[Realtime] 파이프라인 상태 → error
    |
    |- (1) 토스트 알림 (빨간색, 자동 닫힘 안 함 -- 사용자가 반드시 인지)
    |     "파이프라인 오류: 분석 단계 실패"
    |     [상세 보기] [닫기]
    |
    |- (2) HUD에 에러 표시 (빨간 점)
    |
    |- (3) 상단 고정 영역에 에러 상태 표시
    |     타임라인에서 실패 단계 빨간색 ✕
    |     에러 메시지 바로 표시 (아코디언 필요 없이)
    |
    v
[사용자] 토스트 "상세 보기" 또는 활동 패널에서 확인
    |
    v
[활동 패널] 에러 상세 + "다시 시도" 버튼 표시
```

## 4. 분기 (조건)

### 분기 1: 진행 중 파이프라인 유무에 따른 초기 상태

```
[웹앱 접속]
    |
    +-- [진행 중 파이프라인 있음]
    |     |- 활동 패널 자동 오픈
    |     |- 상단 고정 영역에 진행 현황 표시
    |     '- HUD "로그" 버튼에 녹색 점 (활성 표시)
    |
    +-- [진행 중 파이프라인 없음 + 최근 완료 있음]
    |     |- 활동 패널 닫힌 상태
    |     '- HUD에 "로그" 버튼만 표시
    |
    '-- [파이프라인 기록 없음 (첫 사용)]
          |- 활동 패널 닫힌 상태
          '- HUD "로그" 클릭 시 "아직 활동 기록이 없습니다" 표시
```

### 분기 2: 채팅과 활동 패널 동시 열림

```
[채팅 오버레이 열린 상태에서 활동 패널 열기]
    |
    +-- 채팅: 우측 400px
    +-- 활동 패널: 좌측 480px
    '-- 2D 오피스: 가운데 남은 영역

    ※ 화면 폭이 부족하면 (< 1200px):
       활동 패널이 채팅 위에 겹침 (z-index 더 높음)
       활동 패널 닫으면 채팅 다시 보임
```

### 분기 3: 토스트 알림 동작

```
[토스트 발생 조건]
    |
    +-- 파이프라인 시작 (idle → dispatching)
    |     "파이프라인 시작: {task 요약}"  [4초 후 자동 닫힘]
    |
    +-- 단계 전환 (analyzing → planning 등)
    |     "분석 완료. 기획 단계 시작."  [4초 후 자동 닫힘]
    |
    +-- 파이프라인 완료 (→ completed)
    |     "파이프라인 완료: {task 요약}"  [결과 보기] [5초 후 자동 닫힘]
    |
    +-- 에러 발생 (→ error / timeout)
    |     "파이프라인 오류: {단계} 실패"  [상세 보기] [자동 닫힘 안 함]
    |
    '-- 토스트 최대 표시 수: 3개 (초과 시 가장 오래된 것 제거)

[토스트 동작]
    |- 위치: 우상단 (right: 24px, top: 24px)
    |- 쌓임 방향: 아래로
    |- 클릭: 버튼 클릭 시 활동 패널 열림 + 해당 항목 포커스
    |- 마우스 호버: 자동 닫힘 타이머 일시 정지
```

## 5. 예외 처리

### 예외 1: 활동 데이터 로드 실패

| 항목 | 내용 |
|------|------|
| **조건** | hub_pipelines / hub_inter_messages 테이블 쿼리 실패 |
| **감지** | supabase.from() 에러 |
| **UI 동작** | 패널 내 "데이터를 불러올 수 없습니다. 다시 시도" 표시 |
| **복구** | "다시 시도" 클릭 → 재로드 |

### 예외 2: 산출물 파일 로드 실패

| 항목 | 내용 |
|------|------|
| **조건** | /api/pipeline/file 호출 실패 (파일 없음 / 경로 이상) |
| **감지** | API 응답 에러 |
| **UI 동작** | 파일명 아래에 "파일을 불러올 수 없습니다" 인라인 표시 |
| **복구** | 파일명 재클릭 시 재시도 |

### 예외 3: Realtime 구독 끊김

| 항목 | 내용 |
|------|------|
| **조건** | WebSocket 연결 끊김 |
| **감지** | Supabase Realtime channel CHANNEL_ERROR |
| **UI 동작** | (1) 상단 고정 영역에 "실시간 연결 끊김" 배너 (2) 타임라인 진행 멈춤 (stale) |
| **복구** | 자동 재연결 성공 시 → REST로 최신 상태 동기화 + 배너 제거 |

### 예외 4: 동시 파이프라인 (가드레일 위반)

| 항목 | 내용 |
|------|------|
| **조건** | 이미 진행 중인 파이프라인이 있는데 새 트리거 발생 (서버에서 거부) |
| **감지** | 상단 고정 영역에 이미 파이프라인 표시 중 |
| **UI 동작** | 토스트: "이미 진행 중인 파이프라인이 있습니다" |

```
활동 패널 상태:
  closed        -- 패널 닫힘
  open_live     -- 패널 열림 + 진행 중 파이프라인 상단 고정
  open_history  -- 패널 열림 + 진행 중 파이프라인 없음
  open_detail   -- 패널 열림 + 히스토리 항목 상세 확장 중

전이:
  closed → open_live       : HUD "로그" 클릭 (진행 중 파이프라인 있을 때)
  closed → open_live       : 파이프라인 시작 시 자동 오픈
  closed → open_history    : HUD "로그" 클릭 (진행 중 없을 때)
  open_live → open_live    : Realtime 업데이트 (상단 영역 갱신)
  open_live → open_history : 파이프라인 완료 (5초 후 히스토리로 이동)
  open_history → open_detail : 히스토리 항목 클릭
  open_detail → open_history : 상세 접기 또는 다른 항목 클릭
  open_* → closed          : 패널 닫기 버튼 또는 ESC
```
