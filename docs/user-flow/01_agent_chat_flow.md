# 유저플로우 01: 에이전트 대화 플로우

> 관련 PRD: FR-13 (에이전트 클릭 → 채팅), FR-05 (LLM 호출), FR-07 (상태 관리)
> 작성일: 2026-03-27

---

## 1. 개요

사용자가 2D 오피스에서 에이전트 캐릭터를 클릭하고, 채팅 오버레이를 통해 대화하며, 응답을 받아 기록이 저장되는 흐름.

---

## 2. 해피패스 (Happy Path)

```
[사용자] 웹앱 접속
    │
    ▼
[웹앱] 2D 오피스 렌더링 + 에이전트 캐릭터 표시 (idle 상태)
    │
    ▼
[사용자] 에이전트 캐릭터 클릭
    │
    ▼
[웹앱] 캐릭터 선택 하이라이트 + 카메라 패닝 (해당 에이전트 중앙)
    │
    ▼
[웹앱] 채팅 오버레이 열림 (우측, 400px)
    │  ├─ 헤더: 에이전트 이름 + 상태 표시 + 닫기 버튼
    │  ├─ 메시지 영역: 기존 대화 이력 로드 (최근 20건)
    │  └─ 입력바: 텍스트 입력 + 전송 버튼
    │
    ▼
[사용자] 메시지 입력 + 전송 (Enter 또는 버튼 클릭)
    │
    ▼
[프론트] POST /api/chat { message, agent_id, conversation_id }
    │
    ▼
[백엔드]
    ├─ 사용자 메시지 → hub_messages 저장
    ├─ 에이전트 상태 → 'working' 업데이트
    ├─ 이전 대화 20건 로드
    └─ Gemini Flash 스트리밍 호출
    │
    ▼
[프론트] SSE 스트리밍 수신 → 실시간 타이핑 효과로 표시
    │
    ├─ [2D 오피스] 에이전트 캐릭터: idle → working (녹색 펄스)
    ├─ [HUD] "분석관 — 작업 중" 표시
    │
    ▼
[백엔드] 스트리밍 완료
    ├─ 어시스턴트 메시지 → hub_messages 저장
    └─ 에이전트 상태 → 'idle' 업데이트
    │
    ▼
[프론트]
    ├─ 스트리밍 텍스트 → 확정 메시지로 전환
    ├─ [2D 오피스] 에이전트 캐릭터: working → idle (회색)
    └─ [HUD] "분석관 — 대기 중" 복귀
    │
    ▼
[사용자] 대화 계속 또는 채팅 닫기 (ESC / 닫기 버튼 / 빈 영역 클릭)
```

## 3. 분기 (조건)

### 분기 1: 대화 이력이 있는 경우 vs 첫 대화

```
[채팅 오버레이 열림]
    │
    ├── [기존 conversation_id 있음]
    │     └─ hub_messages에서 최근 20건 로드 → 메시지 영역에 표시
    │
    └── [첫 대화 (conversation_id 없음)]
          └─ 빈 메시지 영역 + 에이전트 greeting 표시
               예: "데이터가 뭐라고 하는지 봅시다. 무엇을 분석할까요?"
```

### 분기 2: 다른 에이전트로 전환

```
[채팅 오버레이 열려 있는 상태에서]
    │
    └── [다른 에이전트 캐릭터 클릭]
          ├─ 현재 채팅 오버레이 닫힘 (진행 중이던 스트리밍은 abort)
          ├─ 새 에이전트 선택 + 카메라 패닝
          └─ 새 에이전트 채팅 오버레이 열림
```

### 분기 3: 에이전트가 이미 working 상태

```
[에이전트 캐릭터 클릭 (현재 status: working)]
    │
    ├─ 채팅 오버레이는 정상적으로 열림
    ├─ 헤더에 "작업 중..." 상태 표시
    └─ 메시지 입력 가능 (큐잉)
         └─ 현재 작업 완료 후 새 메시지 처리
              ※ 현재 구현: 동시 요청 허용 (LLM은 stateless)
              ※ 향후 개선: 큐잉 또는 "잠시만 기다려주세요" 안내
```

## 4. 예외 처리

### 예외 1: Empty — 메시지 없이 전송 시도

| 항목 | 내용 |
|------|------|
| **조건** | 사용자가 빈 문자열 또는 공백만 입력하고 전송 |
| **감지** | 프론트엔드에서 trim() 후 빈 문자열 확인 |
| **UI 동작** | 전송 버튼 비활성화 상태 유지. 전송 시도 무시 |
| **복구 동선** | 사용자가 텍스트 입력하면 전송 버튼 활성화 |
| **데이터 영향** | 없음 (API 호출하지 않음) |

### 예외 2: Error — LLM 응답 실패

| 항목 | 내용 |
|------|------|
| **조건** | Gemini API 오류, 네트워크 타임아웃, rate limit 등 |
| **감지** | SSE 스트림에서 `{ error: '응답 생성 중 오류' }` 수신 |
| **UI 동작** | 메시지 영역에 에러 메시지 표시: "응답을 받지 못했어요. 다시 시도해주세요." |
| **에이전트 상태** | `working` → `error` (빨간 점멸) |
| **복구 동선** | (1) 에러 메시지 아래 "다시 시도" 버튼 표시 → 클릭 시 같은 메시지로 재전송 (2) 10초 후 에이전트 상태 자동 `idle` 복귀 |
| **데이터 영향** | 사용자 메시지는 이미 저장됨. 어시스턴트 메시지는 저장되지 않음 |

### 예외 3: Error — SSE 연결 끊김 (스트리밍 도중)

| 항목 | 내용 |
|------|------|
| **조건** | 네트워크 불안정으로 SSE 연결이 중간에 끊김 |
| **감지** | EventSource의 onerror 이벤트 또는 fetch의 ReadableStream 에러 |
| **UI 동작** | (1) 지금까지 수신한 텍스트는 유지 (2) "연결이 끊겼습니다" 인라인 안내 표시 |
| **에이전트 상태** | 백엔드에서 정상 완료 가능 → 상태 불일치 가능 |
| **복구 동선** | (1) "새로고침" 버튼으로 대화 이력 재로드 (2) 에이전트 상태 재조회(GET /api/agents) |
| **데이터 영향** | 백엔드는 정상 처리 가능하므로 메시지 저장 여부 불확실 → 재로드로 확인 |

### 예외 4: Permission — 에이전트 접근 실패

| 항목 | 내용 |
|------|------|
| **조건** | 존재하지 않는 agent_id로 요청, 또는 YAML 파일 누락 |
| **감지** | /api/chat에서 `{ error: 'Agent not found' }` 응답 (404) |
| **UI 동작** | 채팅 오버레이에 "에이전트를 찾을 수 없습니다" 표시 |
| **복구 동선** | 채팅 닫기 → 페이지 새로고침 → 에이전트 목록 재로드 |
| **데이터 영향** | 없음 (DB 기록 없음) |

## 5. 상태 흐름 (State Flow)

```
┌──────────────────────────────────────────┐
│           채팅 오버레이 상태              │
├──────────────────────────────────────────┤
│                                          │
│  [closed] ──클릭──→ [open:idle]          │
│     ▲                    │               │
│     │                    │ 메시지 전송     │
│     │                    ▼               │
│     │              [open:streaming]       │
│     │                    │               │
│     │           ┌───완료──┼───에러──┐     │
│     │           ▼        │        ▼     │
│     │     [open:idle]    │  [open:error] │
│     │           │        │        │     │
│     │           │        │   다시 시도    │
│     │           │        │        │     │
│     │           │        └────────┘     │
│     │           │                        │
│  ESC/닫기───────┘                        │
│                                          │
└──────────────────────────────────────────┘
```

에이전트 상태 전이 (DB):

```
idle ──메시지수신──→ working ──응답완료──→ idle
                         │
                      에러 발생
                         │
                         ▼
                       error ──10초경과──→ idle
```
